## 3. Обучение
В этом проекте используется **LoRA (Low-Rank Adaptation)** для параметро-эффективного дообучения языковой модели. Ниже приведены наиболее важные строки кода, определяющие корректность и качество обучения.

### **3.1. Загрузка базовой модели**
```
base_model = AutoModelForCausalLM.from_pretrained(
    model_name,                    
    torch_dtype=torch.float16,     
    device_map="auto"              
)

```

**Назначение:**
- Загружает **instruction-модель** для генерации текста
- `float16` снижает потребление видеопамяти **в 2 раза**
- `device_map="auto"` автоматически распределяет модель на GPU

### **3.2. Формат обучающих данных**

`{"prompt": "Что такое видеомонтаж?", "response": "процесс создания единого видео из отдельных клипов..."}`

**Назначение:**
- Обучает модель **отвечать на вопросы**
- Формирует связь `вопрос → корректный ответ`
- **Критично** для instruction-моделей (chat/instruct)

### **3.3 Использование LoRA**
```
lora_config = LoraConfig(
    r=32,
    lora_alpha=64,
    target_modules=[
        "q_proj", "k_proj", "v_proj", "o_proj",

    ],
    lora_dropout=0.05,
    bias="none",

```
- Активирует параметро-эффективное обучение без изменения базовых весов модели.

### **3.5. Архитектура LoRA**

**LoRA (Low-Rank Adaptation)** — это способ "доучить" большую модель, не изменяя её полностью.  
Вместо того чтобы снова обучать миллионы весов, LoRA добавляет небольшой дополнительный слой, который корректирует только нужные части уже готовой нейросети.

![LoRA Architecture](https://huggingface.co/datasets/huggingface/documentation-images/resolve/main/peft/lora_diagram.png)
*LoRA добавляет маленькие матрицы B и A только к весам внимания (Q, K, V, O).*

### **3.6. Математическое описание работы с весами**

Обычный слой нейросети выполняет вычисление:
\[
h = W₀ \times x
\]
где  
- **W₀** — основные (замороженные) веса модели,  
- **x** — входной вектор,  
- **h** — результат слоя.

В LoRA этот слой дополняется небольшой поправкой:
\[
h = (W₀ + ΔW) \times x = W₀ \times x + B \times A \times x
\]
где **ΔW = B × A** — это и есть “добавка” LoRA, которая учится во время fine-tuning.



### **3.7. Влияние матриц**

| Матрица | Роль | Что делает |
|----------|------|-------------|
| **A** | Сжимает данные | уменьшает размерность входа до меньшего числа признаков |
| **B** | Расширяет обратно | восстанавливает размер и добавляет нужную поправку |
| **ΔW = B×A** | Маленькое изменение к весам | регулирует работу слоя без перезаписи всех параметров |



### **3.8. Параметры самого обучения**

| Параметр | Значение | Что означает |
|-----------|-----------|---------------|
| **r** | 32 | “ширина” добавки — чем меньше r, тем легче модель |
| **alpha (α)** | 64 | регулирует силу влияния LoRA на основной слой |
| **Dropout** | 0.1 | защита от переобучения |
| **Target modules** | `["q_proj", "k_proj", "v_proj", "o_proj"]` | части блока внимания, где включается LoRA |



Итог:  
LoRA обучает только свои маленькие изменения (**A** и **B**) — а вся остальная модель остаётся прежней.  
Благодаря этому дообучение идёт **в 100 раз быстрее** и требует **в десятки раз меньше памяти**.
